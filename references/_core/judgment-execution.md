# Judgment layer vs execution layer — how the two tiers fit

This system has two tiers, and they map onto the kernel's central line — **判断退守 / 执行充裕**.
Knowing which tier you are on changes what your output *is*.

## The two tiers

- **Architecture / judgment tier** — `ai-native-architect`. It *designs*: where does judgment sit, how
  does context flow, what is the graph, what is the substrate. Output = a **blueprint**. Done occasionally,
  when setting up or re-shaping. It is judgment-heavy meta-work.
- **Execution tier** — the six surface skills (engineering · design · research · learning · innovation ·
  org-operate). Each *does the work*: it is the abundant execution the kernel frees. Output = a **real
  work product** (code+specs+evals, a design+rationale, a research finding+credibility ledger, a learning
  scaffold, an innovation portfolio, an operating runbook). Used continuously, in the doing.

The architect designs the judgment nodes; the execution skills run **between and up to** those nodes,
and must **respect** them. An execution skill never quietly automates a node the architecture reserved
for a human. If you are executing and discover the architecture is wrong (a node is misplaced, a missing
gate), surface it — don't route around it.

## The operating loop every execution skill runs

Within its domain, an execution skill runs the four-step kernel as a loop:

1. **执行充裕 — hand the abundant work to agents.** Generate / transform / traverse / draft at scale.
   This is where AI's leverage is; use it fully. Cheap, parallel, many candidates.
2. **判断退守 — concentrate the human at the few irreducible judgment nodes.** Name them explicitly:
   what's worth doing, choosing among options, owning consequences, signing irreversible/adverse actions.
   Fewer is better; justify every human node you keep. Everything else defaults to an agent.
3. **上下文成基设 — write the result back as compounding context.** Every meaningful step must *feed*
   something retrievable downstream (a decision log, an eval, a knowledge-graph edge, a design token, a
   reflection note). A loop that lives only in prose does not exist. This is what makes the next pass
   smarter and the work *yours*.
4. **人回归意义 — return the freed human attention to meaning.** The point of steps 1–3 is not to remove
   people; it is to free them for judgment, taste, relationship, responsibility, and the question of
   what is worth doing at all.

## 止步线 — the stop-lines (where NOT to offload)

The single most important discipline of the execution layer: **knowing where to stop handing work to
agents.** Abundant execution tempts you to offload past the point where the human capability or the human
stake is the actual value. Each surface has its own stop-lines; the shared rule:

> Do **not** offload the work whose *doing* is the point. Offload the work that is merely *in the way* of
> the point.

- **learning** — never offload the *desirable difficulty*; if the agent does the struggle, the human
  doesn't learn (offloading the answer destroys the very capability the activity exists to build).
- **engineering** — never let an agent's classification *be* the sign-off on an irreversible or adverse
  action; that's a human judgment node.
- **design** — never offload *taste* (the choosing-among-options); generating drafts is abundant, judging
  them is the scarce act.
- **research** — never offload *what is worth knowing* and the final *credibility* verdict.
- **innovation** — never offload *value-perception* (which signal is real, what to back); generating
  candidates is abundant, betting is judgment.
- **org** — never put an agent at the center of a trust/safety/relationship node the architecture reserved
  for a human.

The stop-line is not timidity; it is the kernel's "essence is not maximalism." Crossing it is how you
hollow out the human and call it efficiency.

## A judgment node is a real node, not a label

When you name a human judgment node, make it a *real* gate with teeth:

- It declares its full verdict set — e.g. `{approve, decline, amend}` — and every non-approve verdict
  terminates in a *named* next step (renegotiate / abort / caveated-finding), not a silent stall.
- An **irreversible adverse outcome is itself an irreversible action**: a wrongful reject/fail/deny that
  harms someone belongs behind a human judgment node, not an agent's classification. Design the negative
  path with the same care as the positive one.
- The human's job at the node must be *genuine scrutiny*, surfaced by the observability/eval layer — not
  a rubber stamp. A gate that's theater is worse than no gate.

## Demonstrate, don't attest

Write the deliverable as one practitioner's reasoned work, not a compliance report about itself. Don't
narrate your own conformance ("judgment node here, per the kernel"). The reader should *infer* the rigor
because the work holds together. The kernel and canon should disappear into the work, load-bearing in
substance, not announced.
