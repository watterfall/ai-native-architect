# Self-Test · Are you AI-Native, or AI-Enabled? · 你是 AI-Native 还是 AI 赋能?

> The simplest tool in the box. Pen and paper, ~2 minutes, no AI, no install. It is the scope gate and
> the redraw-vs-graft test made into a card. Answer honestly; the value is in the honest "no."
>
> 工具箱里最简单的一件。纸笔、约 2 分钟、不用 AI、无需安装。它把"范围闸"和"redraw-vs-graft 测试"做成一张卡。
> 诚实作答——价值就在那个诚实的"否"。

---

## The one question that decides everything · 一题定生死

**Delete all the AI from your plan. Does the organization collapse back into a normal org chart with the
same roles and the same hand-offs?**
**把你方案里的 AI 全部删掉。这个组织会塌回成一张普通组织架构图、角色和交接照旧吗?**

- **Yes → you designed AI-enablement, not AI-Native.** AI is bolted onto a pre-AI structure. It will speed
  up individual nodes, but an organization's speed is set by its *graph*, so end-to-end throughput barely
  moves. **是 → 你做的是 AI 赋能,不是 AI-Native。**
- **No → the org cannot exist in that shape without agents. That is the native form.** **否 → 没有 agent 它
  根本不成立——这才是原生形态。**

Everything below just sharpens this answer.

---

## Six checks · 六问 (tick the honest box · 勾诚实那一格)

| # | Check · 检查 | AI-enabled · 赋能 | AI-Native · 原生 |
|---|---|---|---|
| 1 | **The default worker** · 默认工种 | a human does the work; AI assists · 人干活,AI 打下手 | an **agent** is the default; a human is the justified exception · **agent** 默认,人是例外 |
| 2 | **Where humans sit** · 人的位置 | a human waits to approve **every** step · 每一步都等人审批 | a **few** explicit judgment nodes · 少数显式判断节点 |
| 3 | **Knowledge** · 知识 | lives in heads, group chats, slides · 在人脑、群聊、PPT 里 | **compounds** in a system (decision logs, context store) · 在系统里**复利** |
| 4 | **Capacity** · 产能 | more output ⇒ more headcount · 要更多产出就加人 | more output ⇒ more agents + better context, **not** payroll · 加 agent 与上下文,不加人 |
| 5 | **The plan** · 计划 | keep the org chart, add AI on top · 保留架构图,在上面加 AI | redraw the workflow from zero · 从零重画工作流 |
| 6 | **Learning** · 学习 | an occasional retro · 偶尔复盘 | the org **improves itself** via instrumented loops · 通过被观测的回路**自我改进** |

**Mostly the left column?** You are doing (or about to do) **AI-enablement**. That is a legitimate goal,
but it is a *different kind* of thing, and the methodology behind this toolkit is **not for it** in
unmodified form. **多数勾在左边?** 你在做 **AI 赋能**——这是正当目标,但它是**另一种**东西,本方法论原样并**不适用**。

## Which track are you on? · 你在哪条轨?

```
Are you building (or willing to build) an operating structure where agents execute and humans judge?
你是否在建(或愿意建)一个"agent 执行、人来判断"的运作结构?
│
├─ New venture / org / unit, from zero ............................. TRACK A · 绿地新建
│
├─ Incumbent that will carve out an INDEPENDENT new unit, built
│  from zero, run alongside the legacy org .......................... TRACK B · 增量从零切出
│  (the only valid AI-Native path for an existing company · 现有公司唯一有效的原生路径)
│
├─ "Add copilots across our existing departments, keep the chart" ... OUT OF SCOPE · 出域(纯赋能)
│  → You are not the target group of this methodology. Pursue
│    enablement honestly (change management), OR re-enter as Track B.
│    → 你不是本方法论的目标群体。要么诚实做赋能,要么以 Track B 重新进入。
│
└─ Value is led by deep human emotional labor (counseling, hospice,
   the core of childhood education) ................................. BOUNDARY · 边界
   → AI assists, but must not lead. · AI 辅助,不主导。
```

## What to do next · 下一步

- **Track A / B** → fill the [T1 Canvas](t1-canvas.md) (5 minutes), then run the
  [16-Bottleneck Scorecard](bottleneck-scorecard.md). For a full blueprint, paste the
  [portable prompt](portable-prompt.md) into any chatbot, or install the skill (see the repo README).
- **Out of scope** → that is honest, not a failure. Enablement has real value; just hold *node-level*
  expectations, not graph-level ones.
- **Boundary** → design AI around the human core, never through it.

> The methodology this comes from: **organization = distribution of judgment × flow of context.** When
> execution becomes nearly free, judgment is the scarce thing you organize around.
> **组织 = 判断的分布 × 上下文的流动。** 当执行近乎免费,判断才是你要围绕的稀缺品。
