# Lightweight quantitative eval — iteration 1 (with-skill vs baseline)

**Method.** 1 representative eval per execution skill. For each: a *with-skill* agent (reads + follows
`SKILL.md`) and a *baseline* agent (no skill, cold) did the same task and wrote a work product. We
compare the **delta** the skill adds. This is the lightweight pass (user's choice) — not the full
skill-creator grading + browser viewer.

**Honest caveat.** Claude baselines are strong. On several surfaces the baseline independently produced
good work (design clustered directions; research built a credibility scale + pulled real DOIs;
innovation built a de-slop kill funnel). The skill's value is therefore measured as the *consistent
disciplines it forces*, sharpest where the baseline **drifted**.

## Per-surface delta

| surface | baseline did | with-skill delta (what the skill forced) | marginal value |
|---|---|---|---|
| **engineering** | **drifted to an *architecture blueprint*** — it *designed* the system (architect's job) instead of *building* it | kept it a **build** (runnable handler + SPEC + eval suite + JUDGMENT + PERMISSIONS); designed the **adverse path** (wrongful refund *denial* gated like the positive path); froze the recurring bug as a new eval | **high** — corrects a tier drift |
| **learning** | good answer-forward study plan + a stop-line table | reframed **abundant execution as the *temptation***; applied the *constitutiveness* criterion (not just outsourceability) to keep the borrow-checker struggle human; reflection store with named read-back edge | **high** |
| **research** | strong: 5-tier scale, clustered 40→6, real sources | **split the paradigm before grading** (human-lifespan vs mouse/biomarker) so abundant evidence can't answer the wrong question; weak×far claim → **suspend, not delete**; live conflict edges | **high** — prevents a confident-wrong verdict |
| **org** | solid operating model, leaned "blueprint" | the **adverse "no"** (wrongful denial/suspension) gated as its own irreversible action; stop-line holds **under SLA pressure** (escalate up, never auto-resolve); context-rot split into 3 named feeds | **med-high** |
| **design** | strong, domain-anchored, evaluated 3 directions | a genuinely **divergent 5-spread incl. the SaaS-mean generated only to reject as a foil**; 指纹 fingerprint check; soft axes kept *out* of lint; compounding feeds (+ the latent-on-one-off honesty) | **med** (strong baseline) |
| **innovation** | strong de-slop funnel + 70/20/10 | **falsify BEFORE polish**; flagged all 30 inflated on the *viable-path* axis (signal-to-noise collapse) rather than ranking polished one-pagers; bet stays human, who-pays on the same row | **med** (strong baseline) |

## Cross-cutting deltas (present in 6/6 with-skill, largely absent in baseline)

1. **Stays at the execution layer** — produces the actual work product. Baseline engineering & (partly)
   org drifted to *design/blueprint*. The skill keeps you *doing*, not re-designing.
2. **The adverse/negative path designed with equal care** — the single most common "the skill made me do
   this" across all six: gate the wrongful *no* (denied refund, denied customer, suspended weak-but-real
   claim), not just the positive action. Baselines secured the happy path only.
3. **Compounding context as *real artifacts*** — named decision logs, conflict edges, reflection stores
   with a named read-back. Baselines left logs implicit/prose.
4. **The scarce judgment node + stop-line named explicitly** — with-skill always named the one human node
   and what is *never* offloaded; baselines diffused it.

## Conclusion

Over a strong baseline, the skills add consistent, real discipline — most decisively by (a) keeping work
at the execution layer and (b) designing the adverse path + compounding context. The lightweight pass
**supports** the adversarial council PASS (means 8.8–9.0). No skill regressed below baseline on any task.
